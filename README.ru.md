# Kenshi Online - Мультиплеер мод

**Кооперативный мультиплеер для Kenshi до 16 игроков**

Kenshi Online добавляет бесшовный мультиплеер в Kenshi с использованием нативной интеграции MyGUI, сетевого протокола ENet и инжекции плагинов Ogre. Игроки могут вместе исследовать мир, сражаться, строить базы и торговать.

## 🎮 Возможности

- **До 16 игроков** на одном сервере
- **Выделенный сервер** с сохранением прогресса и консольными командами
- **Мастер-сервер** с централизованным браузером серверов (автопоиск)
- **Полная сетевая репликация** - персонажи, NPC, бои, здания, предметы
- **Зональная синхронизация** - эффективное использование трафика
- **Серверная авторитетность** боя и состояния мира
- **Нативный MyGUI интерфейс** - статус-бар, чат с временными метками, список игроков
- **Команды клиента** - `/tp`, `/time`, `/kick`, `/announce`, `/connect`, `/disconnect`, `/pos`, `/players`, `/status`, `/ping`, `/help`
- **Просто запусти и играй** - автоматическая инжекция плагина Ogre, никакой ручной настройки

## 📦 Быстрая установка

### Windows

1. **Скачайте** последнюю версию с [Releases](https://github.com/ваш-username/kenshi-online-simplified/releases)
2. **Распакуйте** архив в любую папку
3. **Запустите** `INSTALL.bat` от имени администратора
4. Установщик автоматически найдет Kenshi (Steam/GOG) и установит мод
5. **Готово!** Запускайте через ярлык "Kenshi Online" на рабочем столе

### Linux (Wine/Proton)

1. **Скачайте** последнюю версию с [Releases](https://github.com/ваш-username/kenshi-online-simplified/releases)
2. **Распакуйте** архив:
   ```bash
   unzip kenshi-online-simplified.zip
   cd kenshi-online-simplified
   ```
3. **Запустите** установщик:
   ```bash
   chmod +x install.sh
   ./install.sh
   ```
4. Установщик автоматически найдет Kenshi и установит мод
5. **Готово!** Запускайте через `~/kenshi-online.sh`

## 🎯 Как играть

### Для игрока

1. Запустите `KenshiMP.Injector.exe` (или ярлык "Kenshi Online")
2. Введите имя игрока и адрес сервера
3. Нажмите **PLAY**
4. Kenshi запустится с включенным мультиплеером
5. В игре нажмите **F1** для открытия меню мультиплеера

### Запуск локального сервера

1. Откройте папку с Kenshi
2. Отредактируйте `server.json` (опционально):
   ```json
   {
     "serverName": "Мой сервер Kenshi",
     "port": 27800,
     "maxPlayers": 16,
     "pvpEnabled": true,
     "gameSpeed": 1.0
   }
   ```
3. Запустите `KenshiMP.Server.exe`
4. Откройте порт **27800 UDP** в роутере/файрволе
5. Игроки подключаются по вашему IP адресу

### Запуск выделенного сервера (VPS)

1. Скопируйте `KenshiMP.Server.exe` и `server.json` на VPS
2. Настройте `server.json`
3. Запустите сервер:
   ```bash
   wine KenshiMP.Server.exe
   # или на Windows:
   KenshiMP.Server.exe
   ```
4. Откройте порт **27800 UDP**

## ⌨️ Управление в игре

| Клавиша | Действие |
|---------|----------|
| **F1** | Открыть/закрыть меню мультиплеера |
| **Insert** | Переключить панель отладки/загрузки |
| **Enter** | Открыть/закрыть чат |
| **Tab** | Переключить список игроков |
| **`** (тильда) | Переключить отладочную информацию |
| **Escape** | Закрыть все панели |

## 💬 Команды чата

В игре откройте чат (Enter) и используйте команды:

- `/connect <ip:port>` - Подключиться к серверу
- `/disconnect` - Отключиться от сервера
- `/players` - Список игроков онлайн
- `/ping` - Показать пинг
- `/pos` - Показать координаты
- `/tp <x> <y> <z>` - Телепортироваться (только админ)
- `/time <час>` - Изменить время суток (только админ)
- `/kick <id>` - Кикнуть игрока (только админ)
- `/status` - Статус сервера
- `/help` - Список команд

## 🖥️ Команды сервера

В консоли сервера:

- `status` - Показать информацию о сервере
- `players` - Список подключенных игроков
- `kick <id>` - Кикнуть игрока
- `say <сообщение>` - Отправить системное сообщение
- `save` - Сохранить состояние мира
- `stop` - Остановить сервер

## 🌐 Сетевой протокол

- **Порт**: 27800 UDP (ENet)
- **Каналы**: 3 (надежный упорядоченный, надежный неупорядоченный, ненадежный последовательный)
- **Тик-рейт**: 20 Hz (50ms)
- **Макс. игроков**: 16

### Синхронизируемое состояние

- Позиции, повороты, анимации персонажей игроков
- Позиции и состояния AI NPC (по зонам)
- Бой: атаки, урон, смерти, нокауты
- Здания: размещение, строительство, разрушение
- Предметы: подбор, выброс, передача в инвентаре
- Время суток, погода, скорость игры
- Сообщения чата

## 🛠️ Архитектура

```
KenshiMP.Injector.exe    -> Модифицирует Plugins_x64.cfg, запускает Kenshi
KenshiMP.Core.dll        -> Загружается Ogre как плагин, перехватывает функции игры
KenshiMP.Server.exe      -> Выделенный сервер (хостинг на VPS или локально)
KenshiMP.MasterServer.exe-> Централизованный браузер серверов (порт 27801)
```

## 🔧 Сборка из исходников (для разработчиков)

### Требования

- **Visual Studio 2022** с C++ Desktop Development
- **CMake 3.20+**
- **vcpkg** (для управления зависимостями)

### Шаги

```bash
# 1. Клонировать с подмодулями
git clone --recursive https://github.com/ваш-username/kenshi-online-simplified.git
cd kenshi-online-simplified

# 2. Установить зависимости через vcpkg
vcpkg install enet:x64-windows
vcpkg install minhook:x64-windows
vcpkg install nlohmann-json:x64-windows
vcpkg install spdlog:x64-windows

# 3. Настроить CMake
cmake -B build -G "Visual Studio 17 2022" -A x64 -DCMAKE_TOOLCHAIN_FILE=[vcpkg root]/scripts/buildsystems/vcpkg.cmake

# 4. Собрать
cmake --build build --config Release

# 5. Файлы в build/bin/Release/
```

## 🐛 Решение проблем

### Мод не загружается

1. Убедитесь, что `KenshiMP.Core.dll` находится в папке с Kenshi
2. Проверьте `Plugins_x64.cfg` - должна быть строка `Plugin=KenshiMP.Core`
3. Запустите игру через `KenshiMP.Injector.exe`, а не напрямую

### Не могу подключиться к серверу

1. Проверьте, что порт **27800 UDP** открыт на сервере
2. Убедитесь, что сервер запущен и работает
3. Проверьте правильность IP адреса
4. Отключите антивирус/файрвол временно для теста

### Игра вылетает при запуске

1. Убедитесь, что у вас установлены все зависимости (Visual C++ Redistributable)
2. Проверьте целостность файлов игры в Steam
3. Удалите мод и установите заново

### Лаги/рассинхронизация

1. Проверьте пинг командой `/ping`
2. Убедитесь, что сервер не перегружен
3. Уменьшите количество игроков или зону синхронизации
